# SafeMind Backend

SafeMind is a FastAPI backend pilot for receiving child-device message events, analyzing psychological signal metrics, building a personal baseline, and deciding when a parent alert should be raised.

The current app is backend + internal evaluation tooling. It does not send real push notifications yet.

## How The System Works

```text
Incoming message
  -> privacy redaction
  -> psychological analyzer
  -> daily metric aggregation
  -> fixed 10-day baseline
  -> deviation detection
  -> alert decision after 3 flagged days in a row
```

Every incoming message is privacy-cleaned and analyzed. The model returns 1-10 scores for:

```text
positive_emotion
negative_emotion
loneliness
anxiety_stress
hopelessness
self_worth_low
risk
```

The DB stores one daily record per child user per calendar day. Multiple messages on the same day are averaged into the same `scores` object.

## Storage Model

MongoDB Atlas is the current working DB for the pilot.

Main collections:

- `daily_signal_scores`: one document per child user per day.
- `user_baselines`: one fixed 10-day baseline vector per child user.

Daily records intentionally do not store:

- raw text,
- redacted text,
- summaries,
- quotes/evidence,
- `daily_score`,
- `baseline_scores`,
- `delta`,
- `score_totals`,
- `pipeline_version`.

The alert reason is generated by code from metric differences, for example:

```text
בדידות +5.4 מהרגיל; חוסר תקווה +4.7 מהרגיל; סיכון +5.7 מהרגיל
```

## Setup

Requires Python 3.11+.

```powershell
python -m venv .venv
.\.venv\Scripts\python.exe -m pip install -e ".[dev]"
Copy-Item .env.example .env
```

Edit `.env`.

For the current pilot flow, use MongoDB:

```env
SAFE_MIND_SIGNAL_STORE_PROVIDER=mongodb
SAFE_MIND_MONGODB_URI=<your MongoDB Atlas connection string>
SAFE_MIND_MONGODB_DATABASE=safe_mind
SAFE_MIND_PSYCHOLOGICAL_ANALYZER_PROVIDER=openai
SAFE_MIND_OPENAI_PSYCHOLOGICAL_ANALYZER_MODEL=gpt-4o-mini
SAFE_MIND_ENABLE_EMBEDDINGS=false
SAFE_MIND_PERSIST_SIGNALS=true
OPENAI_API_KEY=<your OpenAI key>
```

SQLite still exists as a local fallback, but MongoDB is the DB used for pilot testing.

## Eval Dashboard

`GET /eval` opens the internal dashboard.

Use it to:

- run messages through the live pipeline,
- persist synthetic messages,
- list known users,
- inspect alert timelines,
- verify baseline days, flagged days, alert days, and reasons.

This dashboard is internal only.

## Privacy Rules

Do not store raw messages, redacted messages, model summaries, quotes, or evidence phrases.

Only store pseudonymous IDs, dates/timestamps, numeric metric averages, baseline metric averages, alert flags, counts, and deterministic alert reasons.
